01 — Concept

What Is a Tool Here

A practical guide to building interactive AI-powered tools directly inside Claude artifacts — with a live working example.

Platform Tool

Claude invokes these internally — bash, calendar, web search. You can't build these. They're wired into the platform.

HTML Artifact Tool

A self-contained page you build that runs logic, calls APIs, and renders output. Effectively a tool you design yourself.

AI-Powered Artifact

An HTML/React artifact that calls the Anthropic API. Claude inside Claude — the artifact IS the tool AND the intelligence.

03 — JavaScript

API Call Anatomy

JavaScript
const callClaude = async (userInput) => {
  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json"
      // API key injected by platform — never hardcode it
    },
    body: JSON.stringify({
      model: "claude-sonnet-4-20250514",   // Always Sonnet 4
      max_tokens: 1000,
      system: "You are a specialist tool. Respond only in JSON.",
      messages: [
        { role: "user", content: userInput }
      ]
    })
  });

  const data = await response.json();

  // Extract text from content blocks
  return data.content
    .filter(block => block.type === "text")
    .map(block => block.text)
    .join("");
};
model
Always claude-sonnet-4-20250514. Sonnet 4 — fast, capable, cost-effective for artifact use.
max_tokens
Set to 1000. Sufficient for most tool outputs. Raise it if you're generating long structured content.
system
This is where you define the tool's behaviour. Be explicit. "Only respond in JSON." "You are a regex generator." etc.
messages
Array of turns. For a single-shot tool: one user message. For multi-turn: accumulate the full history in state.
API key
Never pass it — the Claude.ai platform injects it automatically. Hardcoding it is both pointless and a security fail.
04 — API Call

Build It Yourself

HTML Skeleton — Copy & Adapt
<!-- Minimal AI tool skeleton -->
<button onclick="run()">Go</button>
<div id="out"></div>

<script>
async function run() {
  const userInput = "your prompt here";

  const res = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      model: "claude-sonnet-4-20250514",
      max_tokens: 1000,
      system: "Your tool's personality and rules go here.",
      messages: [{ role: "user", content: userInput }]
    })
  });

  const data = await res.json();
  const text = data.content.filter(b => b.type==="text").map(b=>b.text).join("");

  document.getElementById("out").textContent = text;
}
</script>

06 — Export

Practical Tips

◆ System prompt is everything. The more specific your system prompt, the more predictable and useful the output. Vague system prompts produce vague tools.
◆ Force JSON output when you need structured data. Tell the model explicitly: "Respond ONLY with valid JSON, no markdown fences, no preamble."
◆ No localStorage in Claude.ai artifacts. Use JS variables and React state instead. Data doesn't persist between page loads here.
◆ Multi-turn? Maintain a history array in state and pass it as the full messages array on each call. The API is stateless — you carry the memory.
◆ React vs HTML. HTML for simple, self-contained tools. React for anything with meaningful state, multiple components, or real-time updates.
◆ Always handle errors. Wrap fetch in try/catch. The API can return error objects — check data.error before accessing data.content.
